Goal:
Build an AI Insights module similar to Microsoft Copilot. The user should be able to ask questions like:

* Show sales trend by item group for the last 6 months
* Which customers declined the most?
* Which product groups are growing?
* Compare current month vs previous month by location
* Which salesperson sold the most tonnage?
* Show customers who stopped buying cement
* Draw a chart of sales by product group
* Show top 10 customers by tonnage
* Explain why sales dropped this month

The AI response should be able to return:

1. A short written insight
2. A chart configuration compatible with Apache ECharts
3. Optional table data
4. Suggested follow-up questions

Important safety rule:
Do not allow the AI to execute arbitrary raw SQL directly.

Instead, create a controlled analytics tool layer. The AI should choose from approved tools only. Each tool should run safe parameterized SQL queries using SQLAlchemy. No DELETE, UPDATE, INSERT, DROP, ALTER, TRUNCATE, CREATE, or arbitrary SQL generated by the model.

5. Create app/services/ai_insights_service.py

Implement a two-step AI flow:

Step 1: Intent planning
Send the user question and available tool descriptions to GPT-5.
Ask GPT-5 to return strict JSON only:
{
"tool_name": "...",
"parameters": {...},
"reason": "..."
}

Allowed tool names only:

* get_sales_trend
* get_sales_by_item_group
* get_sales_by_customer
* get_sales_by_salesperson
* get_sales_by_location
* get_customer_movement_insights
* compare_current_vs_previous_month
* clarify

If the question is unclear, return tool_name "clarify" with a short clarification question.

Validate the chosen tool name and parameters in Python. Never trust the model blindly.

Step 2: Execute approved analytics tool
Run the selected tool function with safe parameterized SQL.

Step 3: Generate final AI response
Send the tool result to GPT-5 and ask it to produce strict JSON matching AIInsightResponse:
{
"answer": "short executive insight",
"chart": {
"type": "bar | line | pie | none",
"title": "...",
"option": {}
},
"table": {
"columns": [],
"rows": []
},
"follow_up_questions": [],
"tool_used": "...",
"assumptions": []
}

Chart requirements:

* Generate Apache ECharts option objects only.
* Use simple, valid ECharts config.
* For trends use line chart.
* For rankings use bar chart.
* For composition/share use pie chart.
* The frontend will render option using echarts-for-react.
* Do not put functions inside the ECharts option because it must be serializable JSON.

Example ECharts option:
{
"title": { "text": "Sales by Product Group" },
"tooltip": { "trigger": "axis" },
"xAxis": { "type": "category", "data": ["Cement", "Roofing"] },
"yAxis": { "type": "value", "name": "Tonnes" },
"series": [
{ "name": "Tonnes", "type": "bar", "data": [120, 85] }
]
}
